# Lab: Lightweight Well-Architected Review

## Objective

Apply the six Well-Architected pillars to a sample workload and produce a small review packet: worksheet, ADR, and improvement backlog.

## Files

- `architecture-review-worksheet.md` - pillar-by-pillar review notes
- `adr-template.md` - decision record for one architecture tradeoff
- `improvement-backlog.md` - prioritized remediation plan

## Steps

1. Define a workload with users, data, availability target, and current deployment model.
2. Complete one review question per Well-Architected pillar.
3. Record one tradeoff in the ADR template.
4. Prioritize at least three improvement backlog items.

## Review Criteria

The packet is complete when each pillar has a finding, every high-risk item has an owner or next step, and the ADR explains the accepted tradeoff rather than only naming a technology.
