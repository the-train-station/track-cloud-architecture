# Lab: AWS Solutions Library Fit Review

## Objective

Turn one AWS Solutions Library reference architecture into a practical review artifact. The goal is not to deploy everything; it is to decide whether the pattern fits a workload and what would need to change before production use.

## Files

- `solution-fit-worksheet.md` - scenario, assumptions, review notes, and backlog
- `cost-performance-tradeoff-table.md` - tradeoff table for architecture choices

## Steps

1. Pick one AWS Solution and write a two-sentence workload scenario.
2. Fill out the solution fit worksheet using the solution diagram and implementation guide.
3. Complete the cost/performance tradeoff table with at least three architecture choices.
4. Add one improvement backlog item that would make the solution safer, cheaper, simpler, or easier to operate.

## Review Criteria

Your lab is complete when the worksheet names the scenario, the selected solution, the key managed services, at least two risks, and one next action. The tradeoff table must explain what you would keep, change, or reject and why.
